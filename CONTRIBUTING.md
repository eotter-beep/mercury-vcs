# Contributing

To contribute to Mercury VCS, you must follow these rules:

- Ran in your own compiler (it can be your favorite, has to support Python, unless your contributing a new file that isnt Python).
- If its a command, follow the naming system Mercury follows, if you dont know, read the [README](https://github.com/eotter-beep/mercury-vcs/blob/master/README.md) commands.
- You may add new commands or features.
- You may use your own coding style if you like.

## PRs

- Say what you changed, you dont need to say why unless its feature removal or a useful add-on.
- Make sure to add a new version in the `version` command via `cli.py`.
